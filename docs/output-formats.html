Output Formats

TSM supports four output formats: table, json, csv, and html.

Table (default)

A UTF-8 box-drawing table suitable for terminal viewing. Complexity values are color-coded: green (<5), yellow (5–9), red (≥10).

JSON

Structured JSON output with the full analysis result, including per-file metrics, per-function metrics, and all threshold violations.

json
{
  "total_files": 2,
  "total_functions": 4,
  "total_loc": 120,
  "files": [
    {
      "path": "src/a.ts",
      "total_loc": 60,
      "total_sloc": 50,
      "function_count": 2,
      "functions": [
        {
          "name": "processData",
          "file": "src/a.ts",
          "line": 5,
          "loc": 55,
          "sloc": 47,
          "cyclomatic_complexity": 11,
          "max_nesting": 4,
          "param_count": 3
        }
      ]
    }
  ],
  "violations": [
    {
      "file": "src/a.ts",
      "entity": "processData",
      "line": 5,
      "metric": "cyclomatic_complexity",
      "value": 11,
      "threshold": 10,
      "severity": "warning"
    }
  ]
}

CSV

RFC 4180-compliant CSV. Header row followed by one row per function. Fields containing commas or quotes are properly quoted.

file,function,line,loc,sloc,complexity,nesting,params src/a.ts,simple,10,15,12,2,1,1 src/b.ts,processData,5,55,47,11,4,3

Violations are not included in CSV output. Use JSON for the full picture.

HTML

A self-contained HTML report with inline CSS and JavaScript. Features:

  • Sortable columns (click any header)
  • Color-coded complexity badges (green/yellow/red)
  • Violations table (if any thresholds exceeded)
  • No external dependencies — single file, works offline
bash
ts-static-analyzer analyze ./src --format html > report.html
open report.html
