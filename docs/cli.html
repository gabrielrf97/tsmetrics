CLI Reference

All flags and options for ts-static-analyzer.

Synopsis

bash
ts-static-analyzer analyze <PATH>... [OPTIONS]

Subcommands

Currently one subcommand: analyze.

Arguments

ArgumentRequiredDescription
<PATH>...YesOne or more file paths or directories to analyze (recursive)

Options

FlagShortDefaultDescription
--format <FORMAT>-ftableOutput format: table, json, csv, html
--verbose-voffReport skipped files to stderr
--min-complexity <N>—noneOnly show functions with CC ≥ N
--min-loc <N>—noneOnly show functions with LOC ≥ N
--timing—offPrint elapsed time and thread count to stderr after analysis
--version——Print version and exit
--help-h—Print help and exit

Examples

bash
# Table output (default)
ts-static-analyzer analyze ./src

# JSON output — pipe to jq for querying
ts-static-analyzer analyze ./src --format json | jq '.files[].functions[] | select(.cyclomatic_complexity > 10)'

# CSV output — import into spreadsheet
ts-static-analyzer analyze ./src --format csv > metrics.csv

# Self-contained HTML report
ts-static-analyzer analyze ./src --format html > report.html

# Filter to complex functions only
ts-static-analyzer analyze ./src --min-complexity 10

# Show timing info
ts-static-analyzer analyze ./src --timing

# Verbose — see skipped files
ts-static-analyzer analyze ./src --verbose 2>&1 | grep SKIP

Exit codes

CodeMeaning
0Success
1Error (parse failure, invalid config, etc.)
Note: When using --min-complexity or --min-loc, the summary counts (files, functions) reflect only the filtered results, not the total analyzed.
